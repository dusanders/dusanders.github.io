Serial Communication

This project started as a learning experience using the MSP430 microcontroller evaluation kit. I wanted to learn the Java implementations available for serial communications. I eventually stumbled on the RXTXcomm libraries that allow for accessing the javax.comm api. The program has a GUI that allows for changing all of the port settings, along with a simple text area to send a message and a text area to view responses.

Building the GUI

I used the Eclipse IDE to build the GUI window, this tool allowed me the ability to 'drag-and-drop' components onto the interface. These components then had to be implemented with the actual calls to the serial communication methods. This is done with setting various listener objects to the components placed on the interface, these listeners will then check what action was requested and then send that specific command. The user is able to view and select the correct comm port number, along with setting the baud rate, data bits, parity bits, stop bits, and flow control to various different configurations.

The data within the window is UART data from the RN4020 Bluetooth Low Energy module from my BTLE RC Car project.

Implement Serial Communications

The application creates an Enumeration list of open serial ports that are detected when it is first launched. Each entry of the list that is returned is checked if is of type Serial, and creates an MspPort object. The newly created MspPort will handle all of calls to the comm port that was passed during the constructor. These MspPort objects are then put into a list which is passed to the GUI class that will display all found Serial ports to the user. 

                
public class JavaMsp {
    public static ArrayList<MspPort> MSP_port_list = new ArrayList<MspPort>();
    
    public static void main(String[] args) {
        //open Enumeration for ports
        Enumeration<?> port_list = CommPortIdentifier.getPortIdentifiers();
        //while we have a list
        while (port_list.hasMoreElements()) {
            //get the element
            CommPortIdentifier port_id = (CommPortIdentifier)port_list.nextElement();
            //got a serial? add it to the list!
            if(port_id.getPortType() == CommPortIdentifier.PORT_SERIAL) {
                MspPort found_port = new MspPort(port_id);
                MSP_port_list.add(found_port);
            }
        }//end while (port_list.hasmoreelemtns
        
        for(int i=0; i < MSP_port_list.size(); i++) {
            System.out.println(i + " : " +MSP_port_list.get(i).get_port_name());
        }
        //lauch the gui
        GUI.main(args);
    }//end main()

}//end JavaMsp

The MspPort class implements the SerialPortEventListener class which is required to handle the callbacks that will be produced by the Serial Port. We also handle all of the opening of the port with the MspPort constructor, while also getting the required input/output streams. 

                
    //open and return an object of MspPort. This is a SerialPort object.
    public MspPort (CommPortIdentifier commPortId) {
        try {
            CommPort openPort = commPortId.open("MspPort", 5000);
            SerialPort serialPort = (SerialPort)openPort;
            input = serialPort.getInputStream();
            output = serialPort.getOutputStream();
            port_name = serialPort.getName();
            serialPort.addEventListener(this);
            serialPort.notifyOnDataAvailable(true);
            serial_port = serialPort;
            connected = true;
            buffer = new String("");
        }//end try 
        catch (PortInUseException | IOException | TooManyListenersException e) {
            e.printStackTrace();
            connected = false;
        }//end catch
    }//end constructor
Here we see the constructor for the MspPort object.

Once the MspPort object is created, its callback is itself. During a callback event the object will attempt to read the data sent and also make a call to the GUI class to update the 'displayInputField' text area which will display the data found at the Serial Port.

Here we see the method that is called when a Serial Port event is received.

The MspPort object also implements two different sending methods to send data to the Serial Port. There is a method that will send a String object; this is done by forming a new byte array by calling 'getBytes()' method of class String. The byte array is then sent to the Serial Port while also checking for Exceptions. Finally, the MspPort is loaded with all of the required 'get' and 'set' methods that are required to change and access the communication settings. 

                 
    //method to send data to serial port
    public boolean sendData(String outputString) {
        byte[] byteString = outputString.getBytes();
                
        try{
            serial_port.setSerialPortParams(baud_rate, data_bits, stop_bits, parity_bits);
            serial_port.setFlowControlMode(flow_control);
            output = serial_port.getOutputStream();
            output.write(byteString);
            output.write(new byte[]{'\n'});
            return true;
        }
        catch (UnsupportedCommOperationException e) {
            e.printStackTrace();
            return false;
        }//end catch Unsupported 
        catch (IOException e) {
            e.printStackTrace();
            return false;
        }
    }
    
    public boolean sendData(int outputInt) {

    }

Conclusion

The application was fun to build, besides the initial research into getting the javax.comm API back into my libraries. The application has helped with a few microcontroller projects that I worked on and performed great. There are a few changes I would like to make; such as, doing away with the default values entered for communication. I would like to set this up so the application does not automatically connect to the Serial Port, but instead waits to connect until all of the settings have been changed. I would also like to see any exceptions thrown to be placed into the text area. Besides a couple changes, I am very happy with how it tunred out, it has helped with both projects listed under my 'Embedded Systems' page; including the programming of the BLTE module I used for my RC car.
